# 1C Repo Map 🗺️

> **Репозиторные карты для конфигураций 1С** — сжатая навигационная структура, которая позволяет LLM (GPT, Claude, Gemini и др.) ориентироваться в больших конфигурациях 1С без чтения тысяч файлов.

---

## ❓ Проблема

Конфигурации 1С часто содержат:
- **10 000–50 000+ файлов** (XML, BSL, макеты)
- **1 000+ справочников**, **500+ документов**, **2 000+ общих модулей**
- Глубокую взаимосвязанность объектов через подсистемы

Когда разработчик просит ИИ «исправить проведение документа Прием», без навигационной карты ИИ:
1. Либо читает десятки нерелевантных файлов
2. Либо теряется в структуре и дает неточные ответы

**Результат:** огромный расход токенов и денег.

---

## ✅ Решение

**1C Repo Map** генерирует навигационные карты в формате Markdown:

| Файл карты | Что содержит |
|------------|-------------|
| `README.md` | Общая статистика конфигурации |
| `objects_index.md` | Алфавитный индекс всех объектов |
| `subsystems.md` | Иерархия подсистем и их состав |
| `modules_graph.md` | Граф зависимостей общих модулей |
| `Catalogs.md` | Структура всех справочников |
| `Documents.md` | Структура всех документов |
| `InformationRegisters.md` | Структура регистров сведений |
| `CommonModules.md` | Карта общих модулей |
| `keywords.md` | Поисковый индекс по ключевым словам |

**Общий размер карт:** ~3–5 МБ (для конфигурации в 1.5 ГБ)  
**Формат:** Markdown — понятен любому LLM без дополнительных инструкций.

## 🚀 Быстрый старт

### 1. Установка

```bash
git clone https://github.com/ВАШ_НИК/1c-repo-map.git
cd 1c-repo-map
pip install -r requirements.txt
```

Требования:
- Python 3.10+
- ~100 МБ RAM на пике
- Доступ к XML-выгрузке конфигурации 1С

### 2. Генерация карт

```bash
python src/generate_repo_map.py \
  --config "/путь/к/выгрузке/1С" \
  --output "./my_config_map" \
  --max-modules 500
```

Параметры:
- `--config` — путь к корневой папке XML-выгрузки конфигурации 1С
- `--output` — куда сохранить карты (создаст папку)
- `--max-modules` — сколько модулей проанализировать на граф зависимостей (по умолчанию 500)

**Время генерации:** 2–5 минут для конфигурации в 50 000 файлов.

### 3. Использование с LLM

При отправке запроса в LLM прикрепляйте карты **перед** исходными файлами:

```
Я работаю с конфигурацией 1С. Вот репозиторная карта:

[вставить README.md]
[вставить Documents.md — если задача про документ]
[вставить subsystems.md — если нужна навигация по подсистеме]

Задача: исправить ошибку проведения документа "Прием".

Конкретные файлы для анализа:
- Documents/Прием/Ext/ObjectModule.bsl
- Documents/Прием/Ext/ManagerModule.bsl
```

## 📊 Эффективность

Тестирование навигации по конфигурации **1С:Здравоохранение72** (~46 000 файлов):

> ⚠️ **Важно:** тест измеряет только токены на **этапе ориентации** в конфигурации (поиск нужных объектов, форм и модулей). Собственно решение задачи (написание/исправление кода) требует примерно одинаковое количество токенов в обоих случаях.

| Задача | Без карты (навигация) | С картой (навигация) | Экономия |
|--------|----------------------|----------------------|----------|
| В документе Направление не заполняется номенклатура в назначенных услугах | 268 185 токенов | 28 293 | **89.5%** |
| В справочнике Сотрудники добавить реквизит для хранения идентификатора ФРМР | 146 145 | 28 247 | **80.7%** |
| В DICOMWorkList не подставляется ScheduledProcedureStepID при формировании Worklist | 261 265 | 28 678 | **89.0%** |
| В документе ВыпискаИзСтационара не формируется CDA при проведении | 182 168 | 28 355 | **84.4%** |
| При получении HL7-сообщения ADT^A08 не обновляются данные пациента | 335 371 | 28 197 | **91.6%** |

**Средняя экономия на этапе навигации: 88% токенов** (~210 000 токенов на задачу).

> Реальная экономия зависит от сложности конфигурации и глубины взаимосвязей. Для узконаправленных задач (найти объект, исправить модуль) выигрыш максимален. Для широких задач (рефакторинг подсистемы) — меньше.

## 🏗️ Архитектура

```
┌─────────────────────────────────────────────┐
│  XML-выгрузка конфигурации 1С               │
│  (Configuration.xml, ConfigDumpInfo.xml)   │
└──────────────┬──────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────┐
│  generate_repo_map.py                       │
│  • Парсит ConfigDumpInfo.xml               │
│  • Сканирует файловую систему              │
│  • Анализирует .bsl модули                 │
│  • Парсит подсистемы                       │
└──────────────┬──────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────┐
│  Markdown-карты (~3-5 МБ)                   │
│  README, индексы, графы, структуры         │
└──────────────┬──────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────┐
│  LLM (GPT, Claude, Gemini и др.)            │
│  Читает карты → находит объект →           │
│  → читает 1-2 исходных файла               │
└─────────────────────────────────────────────┘
```

## 🧪 Тестирование

Для воспроизведения тестов:

```bash
python src/test_tokens.py
```

Скрипт сравнивает два подхода на 5 типичных задачах разработчика 1С и выводит отчет с экономией токенов на этапе навигации по конфигурации.

Результаты сохраняются в `examples/test_results.json`.
